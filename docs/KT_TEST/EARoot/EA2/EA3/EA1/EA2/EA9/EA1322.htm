Een eHealth module is een (native) applicatie en kan op een systeem van de eindgebruiker draaien.  <br/>De eHealth module MOET de 'Client Credentials Grant' flow van OAuth2 (RFC6749) gebruiken om toegang te krijgen tot de gegevensstroom.<br/>Elke eHealth module MOET zich registreren om een (afzonderlijke) unieke client-ID te krijgen.<br/>Een eHealth module registratie KAN worden voltooid door ofwel statische configuratie (out-of-band, via een beheerder, etc) of dynamisch.<br/>Zie <b>RFC 8252</b> voor meer informatie over de implementatie van (native) applicaties waar de browser (gebruiker interface) als losse entiteit wordt beschouwd.<br/>
